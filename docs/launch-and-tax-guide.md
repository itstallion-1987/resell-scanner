# Запуск и налоги — пошаговый гайд (Resell Scanner)

Мастер-runbook: от создания аккаунта Apple до получения денег и уплаты налогов.
Под кейс: соло-разработчик, **налоговый резидент Китая** (свой банк/телефон/налоги в КНР),
приложение продаётся в **США + Европе** (eBay/Vinted/Poshmark/Depop/Mercari), **без Mac**
(сборка на macOS-раннере GitHub Actions). Детали CI-секретов — в `release-pipeline.md`.

> ⚠️ Это операционная инструкция, не налоговая/юридическая консультация. Точные налоговые
> правила Китая, формы и удержания меняются и зависят от твоего статуса — финальные шаги
> по налогам согласовывай с местным бухгалтером.

## Фаза 0 — Подготовить заранее (10 минут)

- [ ] **Apple ID** с включённой двухфакторкой; регион аккаунта — **Китай**.
- [ ] Реквизиты **китайского банковского счёта**: номер счёта, **CNAPS-код** банка, имя владельца
      ровно как в банке (должно совпасть с именем на Developer-аккаунте).
- [ ] **Налоговый номер (TIN)** в Китае и удостоверение личности.
- [ ] **Карта для оплаты** $99/год (членство Apple Developer Program).
- [ ] Решение по типу аккаунта: **Individual** (проще, без D-U-N-S; в App Store как «продавец»
      будет твоё имя) или **Organization** (нужна компания + бесплатный, но небыстрый номер D-U-N-S).
      Для соло — **Individual**.

## Фаза 1 — Apple Developer Program ($99/год)

1. [ ] Зайди на developer.apple.com → **Account** → **Enroll**.
2. [ ] Тип сущности — **Individual / Sole Proprietor**.
3. [ ] Юридическое имя, адрес (Китай), телефон — **должны совпасть** с банком и налоговыми данными.
4. [ ] Оплати $99. Apple может попросить подтверждение личности (фото документа).
5. [ ] Дождись одобрения (обычно 24–48 ч, иногда дольше).
6. [ ] Запомни **Team ID** (Account → Membership) — понадобится для CI (`DEVELOPER_TEAM_ID`).

---

## Фаза 2 — ⚠️ Agreements, Tax, Banking (самое важное для денег)

App Store Connect → **Business** (бывш. «Agreements, Tax, and Banking»). Пока тут не зелёно —
платных продаж и подписок не будет.

1. [ ] **Paid Apps Agreement** — принять. Без него IAP не активируются.
2. [ ] **Banking** → добавить счёт: страна банка **China**, валюта **CNY**, имя владельца = твоё
       (как на аккаунте), номер счёта + CNAPS. Банк должен быть твой (не банк-посредник).
3. [ ] **Tax Forms** — заполнить по подсказкам Apple. Ключевое:
   - **U.S. tax form (W-8BEN)** — т.к. продаёшь в США. Укажи китайский **TIN** и заяви льготу по
     **налоговому соглашению Китай–США** (снижает удержание США с твоих US-продаж). Без формы
     удержание будет максимальным.
   - Прочие региональные налоговые секции — по запросу Apple.
   - Сохрани все налоговые документы Apple: пригодятся, чтобы **зачесть удержанный в США налог**
     против китайского (по соглашению об избежании двойного налогообложения).
4. [ ] Убедись, что все три блока (Agreement / Tax / Banking) в статусе **Active/Complete**.

> 💡 Apple — **merchant of record**: она сама собирает деньги с покупателей, удерживает местный
> НДС/налог с продаж в каждой стране и платит тебе **чистый** доход. Тебе не нужно регистрировать
> НДС в ЕС/США — это забота Apple.

## Фаза 3 — Приложение, подписки, RevenueCat

1. [ ] **Bundle ID**: Certificates, Identifiers & Profiles → Identifiers → создать
       (напр. `com.<твоё>.resellscanner`). Это значение пойдёт в CI как `APP_IDENTIFIER`
       и в `ios/project.yml` (`bundleIdPrefix`).
2. [ ] **New App** в App Store Connect: имя «Resell Scanner: AI Listings», язык English (U.S.),
       категория **Shopping** (вторичная Business), привязать Bundle ID, задать SKU.
3. [ ] **Подписки (auto-renewable)** — группа `Pro`, два продукта:
   - `Pro Monthly` — $6.99
   - `Pro Yearly` — $39.99 (якорь)
   Цены задаются по матрице территорий (выбираешь базовую и проверяешь ЕС-эквиваленты).
4. [ ] ⚠️ **App Store Small Business Program** — подать заявку (Account → Agreements). Если годовой
       доход < $1M, комиссия Apple падает с **30% до 15%**. Для тебя это прямое +15% к выручке —
       не пропусти. (Подписки и так становятся 15% после первого года, программа даёт 15% сразу.)
5. [ ] **RevenueCat**: создать проект → добавить App Store **Shared Secret** → смапить оба продукта
       на entitlement **`pro`** → создать Offering с пакетами annual/monthly.
6. [ ] Вставить **публичный SDK-ключ RevenueCat** в `ios/ResellScanner/Services/PurchaseManager.swift`
       (заменить `REPLACE_REVENUECAT_PUBLIC_API_KEY`).

## Фаза 5 — Подключить релиз к CI (детали в `release-pipeline.md`)

Пайплайн уже написан — нужно один раз отдать ему доступы через **GitHub Secrets**
(Settings → Secrets and variables → Actions). Полный список и как их получить — в
`docs/release-pipeline.md`. Кратко, что понадобится:

- [ ] **App Store Connect API key** (App Store Connect → Users and Access → Integrations → API keys,
      роль App Manager): даёт `ASC_KEY_ID`, `ASC_ISSUER_ID` и файл `.p8` → `ASC_KEY_P8` (base64).
- [ ] `APP_IDENTIFIER` (Bundle ID), `DEVELOPER_TEAM_ID`, `ASC_TEAM_ID`, `APPLE_ID`.
- [ ] **Приватный репозиторий для сертификатов** (fastlane **match**): `MATCH_GIT_URL`,
      `MATCH_PASSWORD`, `MATCH_GIT_BASIC_AUTHORIZATION` (base64 от `x-access-token:<PAT>`).
- [ ] `APP_SHARED_TOKEN` — токен воркера (тот же, что `APP_SHARED_SECRET` в Cloudflare); подставляется
      в релизную сборку, в публичный репозиторий не коммитится.

## Фаза 6 — Первый билд в TestFlight

1. [ ] GitHub → Actions → **Release to TestFlight** → Run workflow → lane **`certs`** (один раз —
       создаёт сертификат и профиль в match-репозитории).
2. [ ] Затем Run workflow → lane **`release`** — CI на macOS соберёт, подпишет и зальёт `.ipa` в TestFlight.
3. [ ] TestFlight обработает билд (~15–60 мин). Добавь себя как **Internal Tester**, поставь TestFlight
       на iPhone, установи сборку.
4. [ ] **Боевой прогон**: фото вещи → генерация (через живой воркер) → экран результата → копирование →
       paywall с **sandbox-покупкой** подписки. Проверь, что Pro разблокирует историю/переключатель платформ.

## Фаза 8 — 💰 Деньги и налоги

**Поток денег:**
1. Покупка подписки → Apple собирает платёж, удерживает местный НДС/налог с продаж.
2. Apple удерживает комиссию: **15%** (если ты в Small Business Program или подписке > 1 года), иначе 30%.
3. Apple платит **чистый доход в CNY** на твой китайский счёт — **раз в месяц**, через ~45 дней после
   конца финансового месяца Apple, при превышении порога **$40**.

**Учёт (веди с первого дня):**
- [ ] Качай **Financial Reports** и **Payments** из App Store Connect ежемесячно — это твои первичные
      документы (валовые продажи по странам, комиссия Apple, удержания).
- [ ] Фиксируй расходы: Anthropic API (~$0.03/объявление), Cloudflare, $99 Apple, RevenueCat. Налогом
      облагается **чистая прибыль**, а не оборот.
- [ ] Храни налоговые документы Apple и **W-8BEN** — для зачёта удержанного в США налога против китайского.

**Налоги в Китае (согласовать с местным бухгалтером):**
- [ ] Доход от Apple — твой облагаемый доход в Китае как налогового резидента. Уточни режим: как
      **физлицо** (IIT) или выгоднее зарегистрировать **个体工商户 / ИП**.
- [ ] Уточни порядок и периодичность деклараций (помесячно/год), ставки и возможные **IT-льготы**.
- [ ] Уточни механику **зачёта** удержанного в США налога по соглашению Китай–США, чтобы не платить дважды.

## Порядок и зависимости (шпаргалка)

```
Apple ID(CN) → Enroll($99) → Paid Apps Agreement + Banking(CN/CNY) + Tax(W-8BEN)
   → Bundle ID → App record → IAP(Pro 6.99/39.99) + Small Business(15%) + RevenueCat
   → Availability(US+EU, без China) → GitHub Secrets → CI: certs → release → TestFlight
   → метаданные + App Privacy → Submit → (после релиза) выплаты CNY → учёт → налоги CN
```

Узкое место всего пути — **Фаза 2** (Banking + Tax). Пройдёт она чисто (имя совпадает, налоговые
формы заполнены) — дальше дойдут штатно.
